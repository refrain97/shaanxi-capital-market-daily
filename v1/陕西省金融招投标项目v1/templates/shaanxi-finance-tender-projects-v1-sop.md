# 陕西省证券公司可投招投标项目数据 V1 制作流程

版本：V1  
适用场景：整理陕西地区证券公司可作为投标人、服务商或联合体成员参与的招投标项目，形成“结构化项目库 + 工作底稿 + 可视化图片 + ima 入库记录”的标准交付。  
首版样例日期：2026-05-11  
首版观察区间：以陕西招标投标公共服务平台当日及近 300 日可检索项目为基础
核心方法：低频高价值项目按“发布主体优先、标题语义复核、正文资格确认、结果回溯更新”处理。

## 1. 交付目标

- 建立可持续增量维护的陕西“证券公司可投”项目数据。
- 每条可投项目保留公告日期、项目阶段、采购人、服务类型、预算或费率、中标金额、投标截止/开标时间、代理机构、地区、投标人资格和来源链接。
- 金额、期限、项目编号等关键字段优先回到招标公告、中标候选人公示或中标结果公告原文。
- 无法确认的字段统一标注“未披露”，不得推算或补写。
- 宁可记录“当日无可投项目”，也不得把金融机构的家具、安保、运维、宣传品等采购误入主表。
- 每期交付必须生成一张适合入库和传播的日报图片，并上传至 IMA 的“公告检索”知识库；上传元数据和文件名需随交付保留。

### 低频更新原则

金融类可投招投标机会更新频率不高，日常默认先走“观察环节”，只有确认新增可投项目、重要待核验线索或结果进展时，才生成完整日报图片并上传 ima。

- 无新增时保留观察记录，并沿用上一期发布模板更新为当日日期，生成当日 PNG 并上传 ima；事实内容和项目库不新增、不改写。
- 模型只阅读新增候选、排除理由、主体观察新增项和上一期累计摘要。
- 不把长窗口平台列表、PDF 全文或历史排除样本整批放入模型上下文。

## 2. 数据口径

### 时间口径

日更默认检索最近 3-7 天；周更复核当年累计项目。平台默认长窗口可使用 `dates=300`，但必须结合标题和正文人工/程序过滤。

### 地区口径

纳入满足任一条件的项目：

- 采购人或分支机构位于陕西省。
- 项目实施地点、服务网点或交付地点位于陕西省。
- 公告明确为陕西省分行、省分公司、陕西辖区或陕西城市项目。

### 证券公司可投口径

纳入：

- 债券发行主承销商、联席主承销商、承销团成员、簿记管理人、债券受托管理人选聘。
- 公司债、企业债、债务融资工具、境外债、ABS/ABN、基础设施 REITs 等资本市场产品相关服务。
- 标准化债券、标准化金融产品、债券投资、固收投资相关的第三方服务机构、投资顾问、投研顾问、交易/风控支持服务；即使标题未出现“承销”也需进入复核池。
- 并购重组财务顾问、资产证券化顾问、融资顾问、上市辅导、改制上市综合服务。
- 明确要求证券公司、证券期货经营机构、投行业务资质或相关牌照的咨询/顾问/承销服务。
- 证券公司可与律所、会计师、评级机构等组成联合体，且公告未排除联合体的资本市场综合服务项目。

暂不纳入：

- 家具、办公用品、宣传品、礼品、印刷品采购。
- 安保、联网报警、消防、物业、保洁、食堂、车辆、档案整理等运营外包。
- 机房值守、IT 运维、设备维保、自助设备、网络设备、系统集成等技术或硬件采购。
- 银行或金融机构作为采购人但采购内容与证券公司业务资质无关的项目。
- 律师、审计、评估等专门牌照服务，除非公告明确证券公司可投或可联合体参与。

## 3. 项目阶段标签

| 标签 | 说明 |
|---|---|
| 招标公告 | 已发布招标公告，尚未披露中标结果 |
| 中标候选人公示 | 已完成评审，处于候选人公示期或候选人阶段 |
| 中标结果公告 | 已披露中标人或成交结果 |
| 待核验线索 | 第三方或宽泛检索命中，但尚未回到官方公告正文确认 |
| 排除样本 | 已确认不属于证券公司可投项目，仅保留排除原因 |

## 5. 来源优先级

1. 招标人官网、银行/邮政集团采购公告。
2. 陕西招标投标公共服务平台/陕西采购与招标网公告页：`http://bulletin.sntba.com/`。
3. 招标代理机构官网。
4. 政府采购网、公共资源交易平台、交易中心。
5. 可信招投标聚合站或财经媒体，仅作为补充线索，入库时标注来源可信度。

### 发布主体观察优先级

陕西地区此类机会发布频率不高，但发布主体通常更集中。每日检索时先建立“主体信号”，再判断标题和正文，不得只按标题关键词一刀切。

优先观察主体：

- 省属国资与省级资本平台：陕西投资集团、陕西金融控股、陕西长安汇通资本、陕西金资、陕西省内其他国资资本运营/基金/投资平台。
- 市属/区县国资平台：西安城投、产投、工投、经发、未央城建、西咸新区发展/沣西发展、雁塔城发等。
- 能源、交通、文旅、产业集团下属资本运作主体：名称含“资本、基金、投资、资产、产投、金控、发展集团、产业集团、城建、城投”的主体。
- 金融机构及其理财、资管、基金、资本子公司：银行、证券、基金、信托、资产管理公司及陕西分支/子公司。

主体分层：

| 层级 | 主体特征 | 处理规则 |
|---|---|---|
| A 重点主体 | 省属/市属国资资本平台、城投产投、金控基金平台 | 标题出现投资、债券、基金、资产、融资、顾问、第三方服务等任一语义即进入待核验 |
| B 关注主体 | 大型国企集团、产业集团、上市公司及其资本运作平台 | 标题出现资本市场或投资服务语义时进入待核验 |
| C 普通主体 | 学校、医院、物业、工程公司、一般采购单位 | 仅在标题或正文明确证券公司可投时纳入 |

主体命中后的排除也要有依据：若 A/B 主体采购家具、保洁、系统运维等普通服务，可排除；若采购内容是“第三方服务机构、咨询服务、管理服务”，但标题同时出现投资、债券、资产、基金、融资等语义，不得在标题层排除。

### 陕西招标投标公共服务平台读取方式

平台首页内嵌列表页：

- 招标公告：`/xxfbcmses/search/bulletin.html`
- 中标候选人公示：`/xxfbcmses/search/candidate.html`
- 中标结果公示：`/xxfbcmses/search/result.html`

常用参数：

- `categoryId=88` 招标公告，`91` 中标候选人公示，`90` 中标结果公示。
- `word` 使用双重 URL 编码关键词，例如“银行”对应 `%25E9%2593%25B6%25E8%25A1%258C`。
- `dates=300` 为平台默认较长时间窗口；日更可用 `dates=7` 或指定 `startcheckDate`、`endcheckDate`。

详情页通常只展示标题、发布日期、媒介和监督部门，正文在 PDF 中。读取正文时：

1. 从详情页 `.mian_list_03` 的 `index` 属性取得公告 ID。
2. POST `http://39.107.102.206:8087/permission/getSecretKey` 获取加密临时密钥。
3. 使用 DES-ECB、key `Ctpsp@884*` 解密后取得 `data`。
4. 请求 `http://39.107.102.206:8087/bulletin/getBulletin/{data}/{公告ID}` 获取 PDF。
5. PDF 多数为扫描件，金额和资格条款应人工/OCR 复核后再入库。

## 7. 检索逻辑

不得只用“债券/承销/证券”单一关键词线性检索。V1 检索必须同时覆盖“主体词 + 产品词 + 服务词 + 语义动作词”，并做结果公告回溯。由于陕西可投机会低频，宁可扩大第一轮候选池，也不要在关键词层漏掉大国企资本运作需求。

### 7.1 主体词

- 资本、金控、基金、投资、资产、产投、城投、城建、发展集团、产业集团、运营管理、国资。
- 陕西、西安、西咸、沣西、雁塔、未央、经发、工投、金资、长安汇通等地域/主体缩写。
- 重点主体全称每周维护一次，新增命中主体要写入观察池。

### 7.2 产品词

- 债券、公司债、企业债、中期票据、短期融资券、超短融、债务融资工具、境外债、海外债、美元债。
- 科创债、绿色债、产业债、可续期债、私募债、定向债务融资工具、PPN。
- 资产证券化、ABS、ABN、CMBS、类 REITs、基础设施 REITs。
- 标准化债券、标准化金融产品、固收产品、一揽子标准化债券、债券投资组合。
- 股权、基金、母基金、产业基金、并购基金、投资基金、引导基金、资产包、存量资产、盘活资产。

### 7.3 服务词

- 主承销商、联席主承销商、承销商、承销团、簿记管理人、受托管理人。
- 财务顾问、融资顾问、投顾、投资顾问、投研服务、研究支持、交易顾问、风险管理服务。
- 第三方服务机构、专业服务机构、中介机构、咨询服务、管理服务；命中时必须结合产品词和采购人判断，不可直接排除。
- 方案设计、尽职调查、估值、定价、投资建议、风险评估、项目筛选、交易执行、存续期管理。

### 7.4 语义动作词

- 发行、注册、申报、承销、募集、选聘、遴选、聘请、采购、咨询、管理、投资、处置、盘活、收购、转让、并购、重组、上市、辅导。
- “聘请第三方服务机构”“选聘专业机构”“咨询服务采购”只有在普通采购语境下才低权重；若与 A/B 主体和投资类产品词同现，必须进入复核。

### 7.5 标题语义判断

标题先按四类处理：

| 标题类型 | 示例语义 | 动作 |
|---|---|---|
| 明确纳入 | 主承销商、债券发行、ABS、财务顾问、上市辅导 | 直接进入候选项目 |
| 重点复核 | 标准化债券、债券投资、基金投资、资产盘活、第三方服务机构、专业服务机构 | 结合发布主体判断，A/B 主体命中则进入待核验 |
| 条件复核 | 咨询服务、管理服务、方案设计、风险评估 | 同时命中投资/债券/基金/资产语义才进入待核验 |
| 明确排除 | 家具、食材、保洁、安保、车辆、工程、设备、运维 | 记录排除样本，不进主表 |

### 7.6 回溯规则

- 对每条招标公告，必须用“项目名称核心词 + 中标结果/中标候选人/成交结果”回查后续公告。
- 对每条结果公告，必须反查“招标公告/项目需求/采购公告”以补齐发布日期、采购内容和资格条件。
- 对标题不含“承销”但含“标准化债券、债券投资、固收、第三方服务机构”的公告，不得在标题层直接排除，先进入待核验线索。

### 7.7 每日执行顺序

日常先执行观察环节：

1. 拉取陕西平台招标公告、候选人公示、中标结果三个栏目最近 3-7 天数据；周更或补漏时使用当年窗口。
2. 先跑主体词，命中 A/B 主体的公告进入主体观察池。
3. 再跑产品词、服务词、语义动作词，形成标题候选池。
4. 将“主体观察池”和“标题候选池”合并去重，做标题级排除。
5. 对剩余项目读取正文或第三方索引，确认采购内容、投标资格、服务边界。
6. 对所有候选项目做结果回溯，补中标候选人、中标人、费率或金额。
7. 输出观察记录：新增可投项目、待核验线索、排除样本和主体观察新增项。
8. 若无新增可投项目、无重要待核验线索、无结果进展，则停止在观察记录，不生成日报图片。
9. 若有增量，再进入正式日报环节：更新项目库，输出 Markdown，生成日报图片，并上传至 IMA 的“公告检索”知识库。

## 8. 图片生成与 IMA 入库

每期日报完成后，必须同步生成图片版并上传 IMA。

### 8.1 图片内容

图片应浓缩 Markdown 日报的核心信息，适合在 IMA 中作为检索入口：

- 标题：`陕西金融类招投标项目清单`。
- 核心指标：当期确认机会数、待核验线索数、已披露候选/中标项目数、跨年延续关注数。
- 主表项目：优先展示项目名称、发布主体、机会类型、项目状态、中标/候选单位或“未披露”。
- 主体观察：展示新增或重点发布主体，尤其是 A 类国资资本平台。
- 漏项/风控提示：列出“标题不含承销但应关注”的语义，如标准化债券、第三方服务机构、投资顾问等。
- 来源说明：保留陕西采购与招标网/陕西招标投标公共服务平台及补充来源名称。

### 8.2 图片规范

- 文件格式使用 PNG。
- 页面优先竖版长图，保证表格字段可读；若项目较多，可拆为主图和明细图。
- 图中不得放未经核验的推断中标单位；未知字段写“未披露”或“待回源核验”。
- 色彩和版式以清晰、稳重为主，避免营销化表达；重点突出“可投机会、主体、阶段、结果”。
- 图片底部必须加入来源标识：`华泰证券西安锦业路证券营业部（西北分公司机构业务中心）` 和 `https://shaanxi-capital-market-daily.vercel.app/v1/`。生成 PNG 后执行：

```bash
python3 v1/scripts/brand_v1_png.py "v1/陕西省金融招投标项目v1/outputs/shaanxi-finance-tender-projects-YYYY-MM-DD.png"
```

上传 ima 前，`v1/scripts/upload_daily_ima.sh` 也会自动补一次。

### 8.3 IMA 上传

知识库名称固定为：`公告检索`。

推荐上传文件名：

`YYYY年M月D日陕西金融类招投标项目清单.png`

本机可用路径：

- IMA skill 根目录：`~/.workbuddy/skills/腾讯ima/`
- 知识库模块：`~/.workbuddy/skills/腾讯ima/knowledge-base/`
- 类型预检脚本：`~/.workbuddy/skills/腾讯ima/knowledge-base/scripts/preflight-check.cjs`
- COS 上传脚本：`~/.workbuddy/skills/腾讯ima/knowledge-base/scripts/cos-upload.cjs`
- API 文档：`~/.workbuddy/skills/腾讯ima/knowledge-base/references/api.md`
- IMA 凭证配置：`~/.config/ima/client_id`、`~/.config/ima/api_key`

凭证读取优先级：

1. 环境变量：`IMA_OPENAPI_CLIENTID`、`IMA_OPENAPI_APIKEY`
2. 本机配置：`~/.config/ima/client_id`、`~/.config/ima/api_key`

上传接口固定走官方 IMA OpenAPI：

- Base URL：`https://ima.qq.com`
- Base Path：`/openapi/wiki/v1`
- 请求方式：`POST`
- Header：`ima-openapi-clientid`、`ima-openapi-apikey`、`Content-Type: application/json`

上传流程：

1. 用 `preflight-check.cjs` 检查 PNG 类型和大小，确认 `media_type=9`、`content_type=image/png`。
2. 调用 `search_knowledge_base` 搜索 `公告检索`，取得 `knowledge_base_id`。当前已验证知识库 ID 为：`WFRa_8s9eTeZR-uXPO4mm9EUAbWmE6RycX6fDtASJeA=`。
3. 调用 `check_repeated_names` 检查推荐上传文件名是否重复。
4. 如同名文件已存在，使用 `_v2`、`_v3` 后缀生成新文件名，不做覆盖或替换。
5. 调用 `create_media` 创建媒体，取得 `media_id` 和 `cos_credential`。
6. 用 `cos-upload.cjs` 将本地 PNG 上传至 COS。
7. 调用 `add_knowledge` 添加入库；`title`、`file_info.file_name` 必须与最终上传文件名完全一致。
8. 调用 `search_knowledge` 做检索验证；至少完整文件名检索命中，才视为上传成功。日期关键词检索也应记录结果。

上传记录 `ima-upload-*.json` 至少记录：

- `uploadStatus`
- `knowledgeBaseName`
- `knowledgeBaseId`
- `uploadedFileName`
- `mediaId`
- `localFile`
- `dataFile`
- `imageFile`
- `verificationQuery`
- `verificationStatus`
- `uploadedAt`
- `notes`

上传完成后，将 `uploadStatus` 写为 `success`，并记录 `knowledgeBaseId`、`mediaId`、`uploadedFileName`、`uploadedAt` 和检索验证结果。如果 API 调用失败，保留 `failed` 或 `draft` 并写明失败环节，不得简单写“没有接口”。

## 9. 增量维护建议

- 每日检索关键词按第 7 节四组词矩阵组合，不再只使用 `债券`、`主承销商`、`承销商` 等窄口径词。
- 每日维护“发布主体观察池”：新增主体、主体层级、命中原因、是否国资/城投/资本平台、历史是否出现可投项目。
- 同一项目按“采购人 + 项目名称 + 招标编号”去重。
- 发现中标候选人或中标结果时，不新增项目，只更新 `stage`、`winningSuppliers`、`winningAmountCny` 和 `progressUpdates`。
- 每周复核一次已过投标截止但尚无结果的项目。
- 命中后先做标题级排除，再读取 PDF 正文确认投标人资格；只有明确证券公司可投或可联合体参与的项目，才进入 `eligibleProjects`。
