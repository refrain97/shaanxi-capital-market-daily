# 陕西上市公司公告早报 V1 制作流程

版本：V1  
适用场景：每日对外发布的陕西辖区上市公司公告早报图片  
固定版式：一页图，HTML/CSS 精确排版后导出 PNG  
默认图片命名：`YYYY年M月D日陕西上市公司早报.png`

## 0. V1 正式发布标准：精读版

自 2026-05-13 起，陕西上市公司公告早报 V1 的正式流程固定为“精读版”，以 2026-05-08、2026-05-11 和 2026-05-13 三版为样例基准。

正式稿不得使用“公告标题自动分类 + 直接堆标题”的方式发布。自动脚本只能用于生成候选清单、发现高价值公告和辅助起草，不能作为最终 Markdown / HTML / PNG。

每日正式稿必须完成以下动作：

1. 逐公司抓取 CNINFO 公告，保留公告总数、覆盖公司数、错误数和日期分布。
2. 按标题和公告类型筛出高价值候选，优先选择风险、监管、诉讼仲裁、质押担保、并购收购、股权激励、员工持股、减持回购、募集资金、经营数据、重大合同、分红和业绩。
3. 下载高价值公告 PDF，抽取文本到 `data/pdf-text-YYYY-MM-DD/`；若 PDF 为扫描件或抽取失败，必须从同日相关进展公告、审计报告或可读附件交叉取数。
4. 从公告原文提取关键数字，不能只引用公告标题。图中每条重点事项应尽量包含金额、比例、股数、价格、期限、覆盖对象、涉案笔数、担保余额、净资产占比、业绩指标等可核验数字。
5. 写一版精读 Markdown：包含“今日一句话”“重点播报”“明日跟踪清单”“播报收尾”。重点播报按 5 月 8 日 / 5 月 11 日格式写：标题为 `公司 代码｜事项`，正文交代事实和数字，最后给出“播报判断”。
6. 手工或半自动生成发布 HTML：六栏仍固定，但栏内内容必须是早报摘要，不是公告标题清单。
7. 导出 PNG、加底部署名、检查图片是否完整展示，之后再上传 ima 和发布网页。

### 精读版六栏内容口径

1. **今日业务机会**：4 个业务主题卡片，每个主题必须写具体公司和数字或明确事项，例如交易风险、诉讼仲裁、并购整合、激励与资本运作。
2. **重大事项与风险公告**：3-4 条最高风险事项，使用 `公司 / 事项 / 业务判断` 三列表。事项列要有关键数字，判断列使用高风险、监管整改、ST观察、损益不确定、高估值等明确标签。
3. **上市公司动态**：4 张横向卡片，展示当天值得业务跟进的动态，不放普通会议公告。
4. **股东变动与资本运作**：正式稿使用 `公司 / 关键数字 / 业务关注` 三列表，不显示公告 ID。这里必须是全图数字最密集的区域。
5. **年报 / 季报 / 业绩固定披露清单**：固定披露只做归集，不抢占重大事项版面。若当天无年报季报，可改为“股东会、业绩说明会与固定披露 / 需延续跟踪的业务线索”。
6. **今日重点跟踪公司**：6 个跟踪对象卡片，写后续跟踪问题，不写“今日披露 X 条公告”这种通用句。

## 1. 固定数据源

### 公司池

使用陕西证监局公开表作为公司池基准：

- 文件：`陕西辖区上市公司基本情况表（2026年3月31日）`
- 样例基准：截至2026年3月31日，有效公司数量为85家
- 样例市场分布：沪市42家、深市37家、北交所6家

日常生成时，陕西上市公司数量不得写死。必须优先检查陕西证监局是否发布新版辖区上市公司表；如有新版，先更新公司池、有效公司数量和市场分布。若未发现新版，再沿用最近一次已核验公司池，并在内部记录使用的基准日期。

### 公告源

主公告源使用巨潮资讯网 CNINFO：

- 公告检索接口：`https://www.cninfo.com.cn/new/hisAnnouncement/query`
- 公告原文：`https://static.cninfo.com.cn/finalpage/...PDF`

必要时交叉核验交易所公告页或公司公告转载，但早报正文以公告原文为准。

## 2. 每日检索口径

### 强制检索原则

日常生成不得只使用 CNINFO 全市场分页结果再做本地过滤。全市场分页在高公告量日期可能只稳定返回每页约30条，且 `pageSize` 表现不稳定，容易漏掉陕西公司公告。

必须使用公司池逐公司检索作为主口径：

- 对 `data/shaanxi-companies-cninfo-2026-03-31.json` 中每家公司逐一请求 CNINFO。
- `stock` 参数必须使用 `证券代码,orgId`，例如 `601225,9900023204`。
- 每家公司独立分页，直到返回数量小于页大小或达到 `totalRecordNum`。
- 用 `announcementId + secCode + announcementTitle` 去重。
- 输出必须记录公告条数、覆盖公司数、按披露日期分布和接口错误数。

推荐脚本：

```bash
python3 scripts/fetch_cninfo_shaanxi_announcements.py \
  --start-date YYYY-MM-DD \
  --end-date YYYY-MM-DD \
  --companies data/shaanxi-companies-cninfo-2026-03-31.json \
  --output data/cninfo-shaanxi-announcements-YYYY-MM-DD.json
```

若脚本输出 `0` 条公告，不得直接写入“无新增公告”结论，必须执行以下二次复核：

1. 对最近一次有公告的日期向前回溯至少5个自然日重新逐公司检索。
2. 随机抽取至少3家高活跃公司（例如隆基绿能、陕西煤业、国际医学、宝钛股份等）单公司查询确认接口返回正常。
3. 用 CNINFO 页面或交易所公告页交叉搜索“陕西 + 日期 + 公告”等关键词。
4. 仅在逐公司检索、回溯检索、抽样单公司检索均无命中后，才允许发布“无新增公告日”。

### 检索日期

使用早报日期作为检索日期，例如：

- 早报日期：`2026年5月6日`
- 图片名称：`2026年5月6日陕西上市公司早报.png`

### 公告窗口

正常交易日：

- 默认检索前一交易日晚间至当日早间披露公告。
- 早报日期当天必须作为检索截止日。

节假日或长周末后：

- 扩大至节前最后一个交易日至早报日。
- 例如五一后早报可用 `4/30-5/6`。

补漏增强规则：

- 每次日报生成时，除主窗口外，应向前回溯至少5个自然日做“高价值公告补漏”。
- 周一早报默认主窗口为上周五晚间/周末至周一，且必须覆盖周六披露日。
- 若前一日生成失败、检索异常或用户反馈遗漏，应回溯至最近一次已确认完整发布日之后，必要时扩大至10个自然日。
- 主图可以聚焦当日和最近窗口，但 Markdown 正文和数据文件必须保留回溯命中清单，便于追溯。

## 3. V1 图片固定栏目

V1 必须保留以下六个栏目：

1. 今日业务机会
2. 重大事项与风险公告
3. 上市公司动态
4. 股东变动与资本运作
5. 年报 / 季报 / 业绩固定披露清单
6. 今日重点跟踪公司

顶部固定信息：

- 标题：`陕西上市公司公告早报`
- 日期：`YYYY年M月D日`
- 覆盖：`覆盖X家上市公司`
- 市场分布：`沪市X家｜深市X家｜北交所X家`
- 来源：`名单来源：陕西证监局｜公告源：巨潮资讯 CNINFO`

底部固定免责声明：

`资料来源：陕西证监局辖区上市公司基本情况表、巨潮资讯公告原文。仅作公告信息整理，不构成投资建议。`

## 4. 内容筛选优先级

### 一级优先：重大风险

优先纳入：

- 证监会立案
- 交易所问询、监管函、关注函
- 监管措施决定书、责令改正、警示函及整改报告
- 独立董事、审计委员会督促函或内部治理异常信号
- 退市风险警示、其他风险警示
- 非标准审计意见、内控否定意见
- 重大诉讼、控股股东股份冻结、债务违约

展示位置：第02栏、第06栏。

### 二级优先：资本运作与股权变动

优先纳入：

- 并购重组、资产出售、竞拍资产
- 向特定对象发行股票、审核问询函回复、募集说明书更新
- 募投项目变更、终止募投、永久补流
- 募集资金现金管理、超授权追认、募集资金临时补流
- 股东减持、权益变动触及1%
- 控股股东或5%以上股东股份质押、解除质押、高比例质押到期
- 回购、股权激励回购注销
- 限制性股票归属、作废、授予价格/数量调整

展示位置：第04栏。

### 三级优先：业务动态

纳入：

- 摘帽、撤销风险警示
- 董监高辞职或变更
- 交易异常波动
- 重大合同、经营数据、业绩说明会

展示位置：第03栏。

### 四级优先：固定时点披露

单独归入第05栏，避免淹没重大事项：

- 年报
- 一季报、半年报、三季报
- 利润分配、分红规划
- 业绩预告、业绩快报

## 5. 数字提取规则

对外发布必须从公告原文提取数字，不能只凭公告标题概括。

数字抽取必须优先从 PDF 原文完成。若本机没有 `pdftotext`，可使用 Python `pypdf` 抽取文本；抽取出的 PDF 文本建议保存到 `data/pdf-text-YYYY-MM-DD/`，便于复核。

### 股东变动与资本运作

必须尽量提取：

- 减持股数
- 减持比例
- 减持均价，如有
- 估算金额，如有均价
- 持股比例变动前后
- 募集资金金额、剩余募集资金金额
- 回购注销股数、回购价格、资金总额
- 质押股数、占所持股份比例、占总股本比例、累计质押比例、到期融资金额
- 担保金额、累计担保余额、占最近一期经审计净资产比例、是否存在逾期担保
- 募集资金现金管理授权额度、实际超额金额、追认时间和是否影响募投项目
- 股权激励归属人数、归属股数、授予/归属价格、股票来源

示例：

`西部材料｜控股股东减持488.20万股，均价57.172元/股，金额约2.79亿元`

### 分红与利润分配

必须尽量提取：

- 每股或每10股派息金额
- 拟派现金红利总额
- 分红基数
- 归母净利润
- 分红比例
- 股权登记日、除息日、发放日，如公告已披露

示例：

`三达膜｜每股派0.21元，拟派6,972.25万元；股权登记日5月11日`

### 季报 / 业绩

优先提取：

- 营业收入
- 同比变动
- 归母净利润
- 同比变动
- 经营活动现金流量净额

示例：

`陕建股份｜营收207.08亿元，同比-10.90%；净利1.78亿元，同比-57.20%；经营现金流-76.39亿元`

## 6. 版式制作规则

### 严谨发布要求

对外发布版本不得使用 AI 图片生成直接生成文字图，因为公司名、金额、比例可能被模型改写。

必须使用：

- HTML/CSS 文本排版
- Chrome headless 截图导出 PNG
- 人工或程序校对数字

### 固定模板

当前 V1 模板文件：

`v1/01-shaanxi-listed-company-morning-report/templates/shaanxi-listed-company-morning-report-v1.template.html`

每日制作时复制模板，替换：

- 日期
- 顶部 KPI
- 六个栏目内容
- 第04、第05栏数字
- 第06栏重点跟踪公司

## 7. 导出图片

推荐尺寸：

- 宽度：1242px
- 高度：2060px

导出命令：

```bash
'/Applications/Google Chrome.app/Contents/MacOS/Google Chrome' \
  --headless \
  --disable-gpu \
  --hide-scrollbars \
  --force-device-scale-factor=1 \
  --window-size=1242,2060 \
  --screenshot=/path/to/YYYY年M月D日陕西上市公司早报.png \
  /path/to/YYYY-MM-DD-html-file.html
```

导出后检查：

```bash
sips -g pixelWidth -g pixelHeight /path/to/YYYY年M月D日陕西上市公司早报.png
```

导出 PNG 后必须加入来源标识，防止图片脱离网页传播后丢失归属：

```bash
python3 v1/scripts/brand_v1_png.py "v1/陕西省上市公司日报v1/outputs/YYYY年M月D日陕西上市公司早报.png"
```

标识内容固定为：`华泰证券西安锦业路证券营业部（西北分公司机构业务中心）` 和 `https://refrain97.github.io/shaanxi-capital-market-daily/v1/`。上传 ima 前，`v1/scripts/upload_daily_ima.sh` 也会自动补一次。

## 8. ima 上传规则

V1 早报生成完成后，必须将发布 PNG 上传到 ima 知识库，并完成检索验证。此步骤是日常交付的一部分，不得只停留在本地生成文件。

目标知识库：

`公告检索`

命名规则：

`YYYY年M月D日陕西上市公司早报.png`

如 ima 已存在同名文件，接口不支持替换，使用：

`YYYY年M月D日陕西上市公司早报_v2.png`

上传流程：

1. `preflight-check.cjs` 检查文件类型
2. `check_repeated_names` 检查重名
3. `create_media` 创建媒体
4. `cos-upload.cjs` 上传文件
5. `add_knowledge` 添加入库
6. `search_knowledge` 检索验证

本机可用脚本位置：

- 类型预检：`~/.workbuddy/skills/腾讯ima/knowledge-base/scripts/preflight-check.cjs`
- COS 上传：`~/.workbuddy/skills/腾讯ima/knowledge-base/scripts/cos-upload.cjs`

凭证来源按以下优先级读取：

1. 环境变量：`IMA_OPENAPI_CLIENTID`、`IMA_OPENAPI_APIKEY`
2. 本机配置：`~/.config/ima/client_id`、`~/.config/ima/api_key`

发布验证要求：

- 上传前必须先通过 `search_knowledge_base` 查找目标知识库 `公告检索`，取得 `knowledge_base_id`。
- 上传前必须用 `check_repeated_names` 查重；如同名文件已存在，使用 `_v2` 后缀重新命名后上传，不执行替换。
- `add_knowledge` 的 `title`、`file_info.file_name` 必须与最终上传 PNG 文件名完全一致。
- 上传后必须用 `search_knowledge` 同时按完整文件名和日期关键词检索；至少完整文件名检索命中后，才视为上传完成。
- 最终回复用户时，需说明 ima 上传是否成功、目标知识库名称、最终入库文件名和检索验证结果。

## 9. 每日交付物

每日建议保留三类文件，并完成 ima 入库：

- 可编辑 HTML：`v1/01-shaanxi-listed-company-morning-report/outputs/YYYY-MM-DD-shaanxi-listed-company-morning.html`
- 发布 PNG：`v1/01-shaanxi-listed-company-morning-report/outputs/YYYY年M月D日陕西上市公司早报.png`
- 文字底稿：`v1/01-shaanxi-listed-company-morning-report/outputs/YYYY-MM-DD-shaanxi-listed-company-morning.md`
- ima 知识库：`公告检索`，文件名同发布 PNG；若重名则使用 `_v2`

## 10. 公司池动态更新规则

每日制作前执行：

1. 检查陕西证监局是否发布新版辖区上市公司基本情况表。
2. 如有新版，下载并统计有效证券代码数量。
3. 按交易所后缀统计市场分布：`.SH`、`.SZ`、`.BJ`。
4. 将顶部“覆盖X家上市公司”和“沪市X｜深市X｜北交所X”替换为最新统计。
5. 若公司简称变化，如撤销ST、变更证券简称，应以公告日最新证券简称为准。
