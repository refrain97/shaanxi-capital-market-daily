# 陕西辖区收并购市场动态看板 V1 制作流程

版本：V1  
适用场景：整理陕西辖区收并购市场动态，形成“工作底稿 + 详细案例长图 + ima 入库”的标准交付。  
首版样例日期：2026-05-07  
首版观察区间：2026-01-01 至 2026-05-07

## 1. 交付目标

V1 目标不是做简单新闻列表，而是形成可继续维护的陕西辖区收并购动态底稿：

- 拉取观察区间内已发生、已披露、正在推进或已终止的陕西相关收并购事件。
- 按四类企业维度归类：已上市公司、拟上市与辅导企业、上市后备企业、活跃非上市公司。
- 提取每个案例的日期、交易方向、交易主体、金额、估值、状态、产业方向、交易意义和后续跟踪点。
- 生成一张精确排版的长图，不使用 AI 生图直接生成文字，避免公司名、金额、比例被改写。
- 将发布 PNG 上传到 ima 知识库并做检索验证。

## 2. 数据口径

### 时间口径

默认从当年 1 月 1 日拉取至制作日。

示例：

`2026-01-01 至 2026-05-07`

### 地区口径

纳入以下任一陕西关联事件：

- 陕西辖区上市公司作为收购方、出售方、重组方或资产处置方。
- 陕西注册/经营主体作为被收购标的、被投资项目公司或交易对手方。
- 外地上市公司收购陕西硬科技、产业链或项目公司资产。
- 陕西上市公司控制或参股项目公司发生股权转让、引资或挂牌处置。

### 事件口径

纳入：

- 股权收购
- 重大资产购买
- 发行股份及支付现金购买资产
- 资产出售或挂牌转让
- 增资入股/合作开发
- 战略控股
- 终止重组
- 停牌筹划重大资产重组

暂不计入主表，但可进入观察池：

- 纯融资事件
- 2025 年发生但 2026 年仍有资本市场热度的历史交易
- 尚无明确收购方或交易动作的潜在线索

## 3. 观察企业四类维度

| 维度 | 定义 | 观察重点 |
|---|---|---|
| 已上市公司 | 陕西辖区 A 股、港股或其他公开市场上市主体及其重要子公司 | 买方、卖方、资产处置、重大资产重组、终止重组 |
| 拟上市与辅导企业 | 曾 IPO 申报、上市辅导、IPO 折戟或明确资产证券化路径的企业 | IPO 转并购、上市公司收购、基金退出 |
| 上市后备企业 | 陕西省级上市后备企业库及 A/B/C 档企业 | 是否与并购事件勾稽、是否形成后备库转并购退出 |
| 活跃非上市公司 | 融资、路演、专精特新、政府基金持股、产业资本关注度高的非上市企业 | 硬科技被收购、产业方控股、项目公司引资 |

## 4. 事件状态标签

| 状态 | 是否计入可量化金额 | 说明 |
|---|---:|---|
| 已完成/工商变更 | 是 | 价款支付、股权交割或工商变更已披露 |
| 已签协议/董事会通过 | 是 | 有明确交易金额，但仍需股东会、监管或交割 |
| 挂牌/推进中 | 是，单独标注 | 有挂牌底价或明确转让底价，最终成交可能变化 |
| 意向/筹划/停牌 | 否 | 暂无最终价格或交易方案 |
| 终止 | 否 | 作为市场动态和失败案例，不计入成交额 |

## 5. 信息源优先级

优先级从高到低：

1. 上市公司公告原文：巨潮资讯、上交所、深交所、北交所、港交所披露易。
2. 监管或交易所披露：证监局、交易所问询、停复牌公告。
3. 可信财经媒体转引公告：证券时报、上海证券报、每日经济新闻、中财网、新浪财经、东方财富等。
4. 公司官网、地方媒体、投融资数据库线索。

发布前，金额、股权比例、估值口径必须尽量回到公告或公告转引来源。无法确认的字段标注“未披露”“待方案”“未最终确定”，不得编造。

## 6. 工作底稿字段

主表字段固定为：

| 字段 | 说明 |
|---|---|
| 序号 | 按首次披露或重大进展日期排序 |
| 日期 | 首次披露日期、草案日期、工商变更日期或终止日期 |
| 陕西关联主体 | 陕西侧上市公司、标的公司或项目公司 |
| 四类维度 | 已上市公司/拟上市与辅导企业/上市后备企业/活跃非上市公司 |
| 交易方向 | 陕西上市公司对外收购、外地上市公司收购陕西标的、资产出售等 |
| 标的/对手方 | 被收购标的、被转让公司、交易对手 |
| 收购方/受让方 | 买方、受让方或拟收购方 |
| 交易类型/轮次 | 现金收购、发行股份+现金、挂牌转让、增资入股等 |
| 股权比例 | 本次交易涉及比例 |
| 金额 | 交易作价、挂牌底价或未披露 |
| 隐含估值/定价口径 | 100%隐含估值、评估值、账面净值、溢价率等 |
| 状态 | 使用第 4 节状态标签 |
| 产业方向 | 军工电子、商业航天、半导体、资源品等 |
| 交易意义 | 对陕西资本市场、产业整合、资产证券化或退出的含义 |
| 后续跟踪 | 股东会、监管审核、业绩承诺、交割、工商变更等 |
| 来源 | 公告或可信来源链接 |

## 7. 文件命名

V1 默认保留以下文件：

- 精简清单：`shaanxi_ma_events_YYYY_ytd.md`
- 详细底稿：`shaanxi_ma_events_YYYY_ytd_detailed.md`
- 制图脚本：`render_shaanxi_ma_cases.py`
- 发布 PNG：`YYYY年M月D日陕西辖区收并购事件详细案例看板.png`
- 工作 PNG：`shaanxi_ma_cases_YYYY_detailed.png`
- 兼容 PNG：`陕西辖区收并购事件详细案例看板.png`，仅用于旧入口或历史链接兼容，不作为 ima 入库标准名。
- 可选 SVG：`shaanxi_ma_cases_YYYY_detailed.svg`

命名规则：

- 从 2026-05-11 起，收并购发布图统一按报告日命名，例如：`2026年5月11日陕西辖区收并购事件详细案例看板.png`。
- 本地输出、入口页链接和 ima 入库默认均使用上述日期文件名。
- 如 ima 已存在同名文件，接口不支持替换，则在日期文件名后追加版本后缀，例如：`2026年5月11日陕西辖区收并购事件详细案例看板_v2.png`。
- 不再使用无日期文件名作为新报告的入库文件名。

## 7.1 Token 友好的增量维护建议

当前 V1 更接近“首版全年底稿 + 确定性渲染脚本”：12 个案例已写入制图脚本，但尚未抽成独立事件库。因此如果每天重新从公开网页检索全年陕西收并购事件，再让模型重新判断、归类、改写，会比证券私募日报更消耗 token，也更容易出现重复案例和口径漂移。

建议升级为 V1.1 的“事件库 + 观察环节 + 有增量再发布”流程：

### 观察环节（默认日常模式）

适用：陕西辖区收并购披露低频、没有必要每天重做全年案例时。

目标是先低成本判断“是否有新增事件或重大进展”，再决定是否生成新看板。

- 每日只检索报告日或最近 3-7 天的新披露、新进展和终止公告。
- 周一、节假日后或公告密集期使用 7 天窗口；普通工作日使用 3 天窗口。
- 只把新增候选网页摘要、公告标题、来源链接和事件库中相关旧案例交给模型。
- 若当天没有新增事件、重大进展、终止公告或金额/状态变化，只记录“观察无新增”，并沿用上一期累计看板模板更新为当日日期，生成当日 PNG 并上传 ima。
- 若只有媒体线索但未回到公告原文，进入“待核验线索”，不进主表，不发布新看板。
- 若出现新增案例、交易状态变化、金额/比例变化或终止重组，再进入正式日报/看板环节。

### 正式日报/看板环节（有增量才触发）

- 更新事件库中的主表案例或 `progressUpdates`。
- 从事件库读取累计案例，生成 Markdown 底稿和 PNG 看板。
- 模型只复核“本次增量 + 受影响旧案例 + 统计摘要”，不重读全部历史案例。
- 发布后生成 ima 上传记录；无新增日期也生成当日日期图片并上传，图片内容沿用累计看板，不新增事实。

### 事件库建议

- 首次运行：把今年以来已确认案例抽成结构化事件库，例如 `data/shaanxi-ma-events-2026.json`，字段沿用第 6 节工作底稿字段。
- 每日运行：只检索报告日或最近 3-7 天的新披露、新进展和终止公告。
- 合并规则：按“陕西关联主体 + 标的/对手方 + 收购方/受让方 + 交易类型”生成事件键；同一事件的新公告只追加 `progressUpdates`，不新增主表案例。
- 出图/日报：从事件库读取累计案例，并只在有新增事件或重大进展时重新生成 PNG；无新增时不重复入库。
- 兜底校验：每周做一次“年初至今关键词 + 公告源”差异检索；每月做一次完整复核，补录遗漏或修正状态。

推荐每日检索范围：

- 必查：巨潮、上交所、深交所、北交所、港交所披露易、陕西证监局。
- 关键词：`陕西 收购`、`西安 收购`、`陕西 重大资产重组`、`西安 股权转让`、`陕西 标的 终止重组`，并叠加重点公司名。
- 时间窗口：日更用 3 天；周一或节假日后用 7 天，避免披露延迟和非交易日堆积。

执行原则：

- 大模型只阅读“新增候选网页摘要/公告要点 + 事件库中的相关旧案例”，不每天重读全部 12 个案例和全年搜索结果。
- 若当天没有新增事件或重大进展，只记录“无新增”，并沿用上一期累计看板模板更新为当日日期，生成当日 PNG 并上传 ima。
- 如只是同一交易的进展公告，更新状态、后续跟踪和来源，不重复计算披露金额。

## 8. 图片制作规则

### 严谨发布要求

对外发布版本不得使用 AI 生图直接生成文字图片，因为公司名、金额、比例、日期容易被模型改写。

必须使用：

- Markdown 或结构化数据作为底稿
- Python/PIL、HTML/CSS、SVG 等确定性排版
- 本地字体渲染
- 人工或程序校对数字

### V1 图片结构

当前 V1 发布图保留两部分：

1. 顶部统计和四类企业维度
2. 逐笔并购案例卡片

第三部分“结构性结论”和“下一步扩表建议”默认不放入发布图。如内部复盘需要，可在脚本中开启，但对外版本保持隐去。

### V1 画布

当前 V1 使用：

- 宽度：1800px
- 高度：3850px
- 背景：浅米白
- 主色：深红、金色、青绿色、石墨黑
- 字体：`/System/Library/Fonts/PingFang.ttc`

### 渲染命令

```bash
python3 v1/陕西省收并购日报v1/scripts/render_shaanxi_ma_cases.py
sips -g pixelWidth -g pixelHeight v1/陕西省收并购日报v1/outputs/shaanxi_ma_cases_2026_detailed.png
sips -g pixelWidth -g pixelHeight v1/陕西省收并购日报v1/outputs/YYYY年M月D日陕西辖区收并购事件详细案例看板.png
```

`render_shaanxi_ma_cases.py` 会在写出 PNG 后自动加入来源标识。若人工重导出或替换 PNG，必须重新执行：

```bash
python3 v1/scripts/brand_v1_png.py "v1/陕西省收并购日报v1/outputs/YYYY年M月D日陕西辖区收并购事件详细案例看板.png"
```

标识内容固定为：`华泰证券西安锦业路证券营业部（西北分公司机构业务中心）` 和 `https://shaanxi-capital-market-daily.vercel.app/v1/`。上传 ima 前，`v1/scripts/upload_daily_ima.sh` 也会自动补一次。

### 打开检查

```bash
open v1/陕西省收并购日报v1/outputs/YYYY年M月D日陕西辖区收并购事件详细案例看板.png
```

检查重点：

- 第三、第四行字段不得挤压或重叠。
- 金额、股权比例、日期和公司名称与底稿一致。
- 长图底部不得残留已隐藏的“第三部分”。
- 标题、统计口径、四类维度、12 个案例均完整显示。

## 9. ima 上传规则

目标知识库：

`公告检索`

默认入库文件名：

`YYYY年M月D日陕西辖区收并购事件详细案例看板.png`

示例：

`2026年5月11日陕西辖区收并购事件详细案例看板.png`

如 ima 已存在同名文件，接口不支持替换，使用：

`YYYY年M月D日陕西辖区收并购事件详细案例看板_v2.png`

示例：

`2026年5月11日陕西辖区收并购事件详细案例看板_v2.png`

### 本机脚本

- 统一上传入口：`v1/scripts/upload_ima_png.cjs`
- ima API：`~/.workbuddy/skills-marketplace/skills/ima-skills/ima_api.cjs`
- 类型预检：`~/.workbuddy/skills/腾讯ima/knowledge-base/scripts/preflight-check.cjs`
- COS 上传：`~/.workbuddy/skills/腾讯ima/knowledge-base/scripts/cos-upload.cjs`

### 凭证来源

按优先级读取：

1. 环境变量：`IMA_OPENAPI_CLIENTID`、`IMA_OPENAPI_APIKEY`
2. 本机配置：`~/.config/ima/client_id`、`~/.config/ima/api_key`

### 上传流程

1. `search_knowledge_base` 查找目标知识库。
2. `preflight-check.cjs` 检查 PNG 类型和大小。
3. `check_repeated_names` 检查同名文件。
4. `create_media` 创建媒体并取得 COS 上传凭证。
5. `cos-upload.cjs` 上传文件到 COS。
6. `add_knowledge` 添加到知识库。
7. `search_knowledge` 用完整文件名和关键词检索验证。

注意：当前 ima 知识库接口返回字段可能是 `kb_name/kb_id`，不要只按旧文档中的 `name/id` 解析。

### 验证口径

至少完成两类检索：

- 完整文件名：`YYYY年M月D日陕西辖区收并购事件详细案例看板.png`
- 关键词：`陕西辖区收并购`

二者均命中后，视为上传成功。
